# Vision And Scope

## ภาพรวม

PR-OS คือระบบบริหารจัดการงานประชาสัมพันธ์สำหรับใช้ภายในสำนักงานเทศบาล จุดตั้งต้นของระบบคือการย้ายข้อมูลจากกระดาน/ไวท์บอร์ดและการสื่อสารกระจัดกระจาย ให้มาอยู่ในระบบเดียวที่มองเห็นง่าย ตรวจสอบได้ และส่งต่อข้อมูลถึงผู้ปฏิบัติงานได้ทันเวลา

ระบบนี้ไม่ใช่เว็บประชาสัมพันธ์สาธารณะ แต่เป็น operational system สำหรับคนทำงานจริงในฝ่ายประชาสัมพันธ์ หัวหน้างาน ผู้ได้รับมอบหมาย และผู้บริหาร

## เป้าหมายระยะยาว

- ลดการถามซ้ำว่า วันนี้มีงานอะไร ใครไปที่ไหน และใครรับทราบแล้ว
- ลดความเสี่ยงงานตกหล่นจากหลายช่องทางรับเรื่อง เช่น หนังสือราชการ LINE โทรศัพท์ และการแจ้งปากเปล่า
- เก็บหลักฐานการแก้ไขงานอย่างเป็นระบบ เพื่อย้อนดูได้ว่าใครเปลี่ยนอะไร เมื่อไร
- ทำให้การมอบหมายงานและการรับทราบงานเป็น traceable workflow
- สร้างรายงานผู้บริหารแบบรวดเร็ว อิงข้อมูลจริง และตรวจสอบที่มาได้

## Phase 0: Developer Handoff

ผลลัพธ์ของ Phase 0 คือชุดเอกสารและโครงโปรเจกต์สำหรับส่งต่อโปรแกรมเมอร์ ประกอบด้วย:

- PRD และ requirement ที่สรุปจากบทสัมภาษณ์
- user flow ของแต่ละบทบาท
- data model และ schema draft
- API contract
- UI screen specification
- Next.js mock screens พร้อม mock data ภาษาไทย
- implementation plan และ checklist สำหรับเริ่มพัฒนาจริง

Phase 0 ยังไม่ใช่ระบบ production และไม่มีการเชื่อมต่อฐานข้อมูลจริง

## Phase 1: MVP ใช้งานจริง

MVP ควรมีความสามารถหลัก:

- login และ role พื้นฐาน
- เพิ่ม แก้ไข เผยแพร่ ยกเลิกงานประชาสัมพันธ์
- ตารางงานและตัวกรอง
- จอมอนิเตอร์แสดงงานวันนี้ พรุ่งนี้ และ 7 วันข้างหน้า
- มอบหมายบุคคลและบทบาท
- รับทราบงานผ่าน responsive web
- แจ้งเตือนผ่าน LINE พร้อม Email fallback
- audit log สำหรับการเปลี่ยนแปลงสำคัญ
- รายงานพื้นฐานและ export PDF/Excel

## Phase 2: Reporting And Operations

- dashboard สำหรับหัวหน้างานและผู้บริหาร
- workload report แยกตามบุคคล บทบาท หน่วยงาน และช่วงเวลา
- report template สำหรับรายวัน รายสัปดาห์ รายเดือน
- notification quota dashboard
- settings สำหรับ master data และ notification template

## Phase 3: Automation And Integration

- OCR หนังสือราชการหรือกำหนดการ
- import งานจากระบบสารบรรณหรือช่องทางที่มี API
- AI ช่วยสรุปเอกสารและตรวจข้อมูลซ้ำ
- BI integration หรือ data warehouse หากปริมาณข้อมูลมากขึ้น
- native app เฉพาะกรณี responsive web ไม่เพียงพอ

## Out Of Scope สำหรับ MVP

- การอ่าน LINE group อัตโนมัติ
- AI วิเคราะห์เอกสารหรือข้อความแบบ production
- OCR เอกสารราชการ
- native mobile app
- public event website
- workflow อนุมัติหลายชั้นแบบซับซ้อน
- integration กับระบบสารบรรณจริง

## หลักออกแบบ

- ใช้งานง่ายกว่ากระดานเดิม ไม่เพิ่มภาระให้เจ้าหน้าที่
- หน้าจอ backend ต้องเร็ว ชัด และกรอกงานได้ใน 1-2 นาที
- จอมอนิเตอร์ต้องอ่านง่ายจากระยะไกลและไม่แสดงข้อมูลส่วนตัว
- ระบบต้องรองรับการเพิ่ม field, report, role และ notification channel ภายหลัง
- ข้อมูลสำคัญต้องตรวจสอบย้อนหลังได้
